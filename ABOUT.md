# 가즈아 (GAZUA) — 주식 자동매매 에이전틱 AI 시스템

> **투자팀을 고용하지 마세요, 배포하세요.**
> 애널리스트 패널, 리스크 데스크, PM, 그리고 반대 심문까지 — 사람 운용팀의 의사결정 구조를 그대로 옮긴 **7-에이전트 AI 매매 팀**입니다.

가즈아는 한국 주식(KOSPI) 시장을 대상으로, **레짐(시장 국면) 판정 → 자산배분 → AI 매매 팀 검토·결정 → 하드 한도 강제 → 집행**을 매 거래일 1회 무인으로 수행하는 자동매매 엔진입니다. 한국투자증권(KIS) Open API와 직접 연동하며, **모의투자 모드가 기본값**입니다.

이 문서는 [수익률 대시보드](https://taehun.github.io/gazua-dashboard/)에서 운용되는 가즈아 엔진의 기술 개요·특징·장단점을 정리한 소개 자료입니다.

---

## 1. 핵심 철학 — 결정론 코어 + LLM 감독

가즈아의 설계 원칙은 **"수치는 코드가, 판단은 AI가, 안전은 다시 코드가"** 입니다.

- **결정론 코어** — 시그널 계산 → 레짐 판정 → 리스크 한도 → 포트폴리오 비중 산출은 전부 순수 퀀트 로직(재현 가능한 함수)이 담당합니다.
- **LLM 매매 팀** — 7개의 LLM 에이전트가 사람 운용팀처럼 검토·토론·결정합니다.
- **하드 한도(enforce_limits)** — AI가 무엇을 결정하든, 포지션·레버리지·집중도 한도는 **코드가 최종 강제**합니다. 프롬프트로는 안전장치를 풀 수 없습니다.

> 백테스트와 실거래가 **동일 코드 경로**를 사용하고, "같은 입력 → 같은 주문" 회귀 테스트로 고정됩니다.

---

## 2. AI-FIRST — PM이 제1 의사결정자

`llm.role: manager` (기본값)에서는 **AI 포트폴리오 매니저가 매일 목표 비중을 직접 산출**합니다. 퀀트 후보와 레짐은 *참고 의견*으로만 입력되며, "승인만 하고 넘어가는" 탈출구가 없습니다. 알파 결정은 AI가, 규칙은 리스크 데스크만 담당합니다.

`llm.role: advisor` (보수 모드)로 설정 한 줄만 바꾸면 결정론 후보를 검토·승인하는 룰 중심 구성으로 즉시 전환됩니다.

**성과는 데이터가 심판합니다.** 매 사이클 *AI 결정 vs 결정론 기준선*의 기여를 누적 측정(`ai_contribution`)하며, 기여가 음수로 정착하면 자동으로 advisor로 복귀합니다 — AI에 대한 신뢰는 측정 가능해야 하니까요.

## 3. 7-에이전트 매매 팀

운용사의 조직도를 그대로 코드로 옮겼습니다.

| 사람 팀의 역할 | 에이전트 | 책임 |
|---|---|---|
| 시장 애널리스트 | **MarketAnalyst** | T-1 시장·기술적 해석, 레짐 판단 동의/이의 |
| 기업분석 애널리스트 | **FundamentalAnalyst** | 공시·수급·밸류에이션 맥락 평가 |
| 매크로 이코노미스트 | **MacroAnalyst** | 미국 증시·환율·유가·VIX 파급 해석 |
| 리스크 데스크 | **RiskManager** | 한도·집중·레버리지 우려 검증 |
| 포트폴리오 매니저 | **PortfolioManager** | 의견 종합 — 매일 목표 비중을 직접 결정 |
| 투자심의위원회 | **Reflection** | PM 결정 반박 시도 — 승인 / 수정안 / 기각 |
| 트레이딩 데스크 | **ExecutionAgent** | 주문 집행·체결 추적 (결정론 상태기계) |

> 사람 팀과 다른 점 — **매일 출근하고, 감정이 없고, 모든 발언이 기록됩니다.**

**협업 흐름**: `Router(패널 소집)` → `애널리스트 3인 병렬 분석`(개별 실패는 결석 처리) → `RiskManager 검증` → `PortfolioManager 결정` → `Reflection 반박 심문`(수정안도 게이트를 다시 통과) → `자율성 게이트 → enforce_limits 하드 한도 → ExecutionAgent → KIS Open API`.

## 4. 팀 프로토콜 — 사람 팀의 규율을 코드로 강제

- **T-1 스냅샷** — 모든 에이전트는 *전일 종가 기준 스냅샷*만 봅니다. 실시간 시세는 비중 결정에 유입되지 않아 **모든 결정이 재현 가능**합니다 (look-ahead 차단).
- **레짐 로스터** — 라우터가 시장 국면별로 패널을 선발합니다. 방어 국면에는 축소 패널로 신속하게 움직입니다.
- **Fail-soft** — 애널리스트 개별 장애는 해당 시각만 제외하고, 의사결정 경로 장애는 결정론 폴백으로 처리합니다. **LLM은 단일 장애점이 아닙니다.**
- **비용 통제** — 운영 기본 ~5 LLM 콜/일. structured output과 backoff 재시도로 호출 신뢰성을 확보합니다.

## 5. 리스크 데스크 — AI가 무엇을 결정하든 안전장치는 코드

| 장치 | 내용 |
|---|---|
| **자율성 게이트** | 3단계 — conservative는 위험 축소만 수용, 노출 확대는 한도 내에서만 |
| **하드 한도** | 포지션·인버스 캡을 LLM 출력 위에 코드로 최종 강제 |
| **킬 스위치** | 장중 5분 간격 감시 — 손실 한도 위반 시 자동 분할 청산 |
| **세이프 홀드** | 데이터 무결성 실패·브로커 장애 시 신규 매매 중지 |
| **실거래 삼중 잠금** | 설정 2곳 + 환경변수가 모두 일치해야만 실거래가 켜짐 |
| **감사 로그** | git sha · 설정 해시 · 전 에이전트 발언까지 — 모든 결정 완전 재현 |

## 6. 검증 — 결정론 기준선은 실데이터, AI 기여는 전방 A/B

엔진의 **결정론 기준선(off 모드)** 을 레퍼런스 전략으로 9.5년 실데이터(KRX) 워크포워드 검증한 결과입니다.

| 지표 | 값 |
|---|---|
| CAGR (거래비용 포함) | **8.59%** |
| Sharpe Ratio | **0.93** |
| Max Drawdown | **-18.3%** |
| 검증 거래일 (2017–2026) | **2,309일** |

- **과최적화 게이트 통과** — 학습 구간(2017–2022) Sharpe 0.56 → 검증 구간(2023–2026) 1.03. 슬리피지 2배 스트레스에서도 기준 유지.
- **AI 성과를 백테스트로 주장하지 않습니다** — LLM은 과거 시장을 이미 학습했기에(사전학습 누출) 백테스트가 무의미합니다. AI 팀의 기여는 **전방 A/B 기록으로만** 측정합니다.

> 위 수치는 레퍼런스(결정론) 전략의 백테스트 결과이며, 실제 운용 성과는 탑재 전략과 설정에 따라 달라집니다. 최신 운용 지표는 [대시보드](https://taehun.github.io/gazua-dashboard/)에서 확인하세요.

## 7. 기술 스택

| 계층 | 구성 |
|---|---|
| **Runtime** | Python 3.12+ · FastAPI (async-first) · uv |
| **Agents** | LangGraph 1.0 StateGraph · Claude / OpenAI |
| **State** | PostgreSQL — 거래 상태 · 감사 로그 · 전방 A/B 기록 |
| **Broker** | 한국투자증권(KIS) Open API 직접 REST — 모의/실전 동일 API |
| **Scheduler** | APScheduler — KST 장 운영, KRX 휴장일 캘린더 |
| **Deploy** | `docker compose up -d` 하나로 DB·API 전체 스택 기동, 단일 서버 무인 운영 |

**엔지니어링 품질**: 4계층 테스트(골든·속성·계약·동시성) + 팀 거버넌스 테스트로 회귀를 차단합니다. 주문은 `cycle_id` 멱등성 키로 중복을 차단하고, 단일 워커 강제 + DB 행 잠금으로 스케줄 중복 실행을 방지합니다.

전략·자율성·프롬프트까지 **전부 `strategy.yaml` 한 파일** — 코드 수정 없이 팀의 성격을 바꿀 수 있습니다.

## 8. 장점 / 강점

- ✅ **재현 가능성** — T-1 스냅샷 고정 + 결정론 코어로 "같은 입력 → 같은 주문"을 보장. 모든 결정을 사후 완전 재현.
- ✅ **다층 안전장치** — AI 일탈을 코드가 막는 하드 한도, 킬 스위치, 세이프 홀드, 실거래 삼중 잠금.
- ✅ **단일 장애점 없음** — LLM 장애 시 결정론 폴백으로 매매가 멈추지 않음.
- ✅ **측정 가능한 AI 신뢰** — AI 기여를 전방 A/B로 누적 측정, 음수 정착 시 자동 강등.
- ✅ **운영 단순성** — 서버 한 대 + `docker compose`로 무인 운영, 설정은 YAML 한 파일.
- ✅ **투명성** — git sha·설정 해시·에이전트 발언까지 남기는 완전 감사 로그.

## 9. 한계 / 주의점

- ⚠️ **시장 위험** — 자동매매는 손실 가능성을 제거하지 않습니다. Max Drawdown이 실재하며, 레버리지·커버드콜 ETF는 변동성이 큽니다.
- ⚠️ **단일 시장·전략 의존** — 현재 전략은 국내(KOSPI) ETF 코어 중심입니다. 개별주 슬리브·해외·옵션 확장은 알파 검증 후 점진 도입 예정이며, 미검증 팩터는 **기동 게이트로 차단**됩니다.
- ⚠️ **LLM 비용·지연** — AI 팀 운용은 LLM 호출 비용과 외부 API 의존을 수반합니다(콜 수·모델은 설정으로 통제).
- ⚠️ **데이터 소스 의존** — 시세·매크로 피드의 신선도/지연이 신호 품질에 영향을 줄 수 있어, 무결성 실패 시 보수적으로 세이프 홀드합니다.
- ⚠️ **백테스트 ≠ 미래** — 검증 수치는 레퍼런스 전략의 과거 성과이며, 미래 수익을 보장하지 않습니다.
